# Security Auditing and Intrusion Detection (FoodDash)

## 1. Database Schema (SQL)

Full SQL schema is available in:
- `database/security_audit_schema.sql`

Core tables:
- `audit_logs`: immutable-style audit events with encrypted metadata and chained record hashes.
- `intrusion_alerts`: threat alerts generated by IDS rules.
- `blocked_ips`: temporary or manual IP blocks.

## 3. Detection Logic (Sample)

### Failed login threshold
Rule:
- If `failed_login` attempts from same IP reach `5` in `60` seconds:
  - Create `intrusion_alerts` record (`failed_login_threshold`, severity `critical`)
  - Block IP in `blocked_ips` for `900` seconds

Configured in:
- `app/Config/IntrusionDetection.php`

### CAPTCHA trigger
Rule:
- If failed attempts from same IP reach `3` within `5` minutes:
  - Require CAPTCHA challenge on login page

### Unauthorized access
Rule:
- If `unauthorized_access` attempts reach `3` in `60` seconds:
  - Create high severity alert

### Suspicious account behavior
Rule:
- On successful login, compare recent token records in short window.
- If IP/device differs significantly in the window:
  - Create `location_or_device_anomaly` alert
  - Log `suspicious_account_behavior`

## 4. Alerting and Notifications

Admin dashboard:
- Security monitor page now shows:
  - Threat metrics
  - Recent intrusion alerts
  - Active blocked IP entries

Email alerts (optional):
- Set `adminAlertEmail` in `app/Config/IntrusionDetection.php`
- Alert emails are sent when IDS raises an alert.

## 5. Audit Report Generation

### Web download
- CSV: `/dashboard/admin/security/report?period=daily&format=csv`
- PDF: `/dashboard/admin/security/report?period=weekly&format=pdf`
- JSON: `/dashboard/admin/security/report?period=monthly&format=json`

### CLI periodic reports
Command:
- `php spark security:report --period=daily`

Output:
- `writable/reports/security/security_report_<period>_<timestamp>.csv`

## 6. Security Controls Applied

- Prepared statements via CodeIgniter Query Builder.
- Sensitive event metadata encrypted using `SensitiveDataService`.
- IP/user-agent data indexed by irreversible hash values.
- Audit log tamper-resistance via chained hashes (`prev_hash`, `record_hash`).
- Access to security data restricted to admin dashboard route checks.
